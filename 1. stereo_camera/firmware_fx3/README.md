## Firmware (Cypress EZ-USB FX3)

Base: [`FeeLab/cyfxuvc_an75779`](https://github.com/FeeLab/cyfxuvc_an75779) — based on Cypress AN75779 *"Interfacing an Image Sensor to EZ-USB FX3 in a UVC Framework"*.

### Power-Up Sequence

#### Initial State (T = 0)

Ensure all FPGA/MCU pins are in safe states before powering rails:

| Signal | State |
|---|---|
| Power rails | OFF (0 V) |
| Crystal Enable | LOW (oscillator off) |
| XCLR (Reset) | LOW (hold reset) |
| XVS | Low or High-Z |
| I2C Bus | High-Z (resistor pull-ups) |

#### Step-by-Step Sequence

**Step 1 — Power rails ON**
```
Order: Analog 2.8 V → Interface 1.8 V → Digital Core 1.2 V
Wait:  5–10 ms for voltages to stabilise
```

**Step 2 — Enable Input Clock (INCK)**

Sony sensors require INCK running *before* XCLR is released.
```
Action: Crystal Enable → HIGH
Wait:   ~5 ms  (crystal warm-up time)
Note:   If the FPGA generates the clock internally, enable the FPGA clock output block here instead.
```

**Step 3 — Release Reset (XCLR)**
```
Action: XCLR → HIGH (1.8 V)
Note:   Raise XCLR for both sensors simultaneously (they are on separate I2C buses).
Wait:   ~1 ms for internal sensor logic to initialise
```

**Step 4 — I2C Configuration**

Send I2C register writes to both sensors to set resolution, frame rate, and streaming mode (see [IMX477 Register Configuration](#imx477-register-configuration)).

> Reset trigger originates from the FX3, which in turn is triggered from the host PC over USB.

### USB3 SDK Setup

Key files modified from the base example:

| File | Purpose |
|---|---|
| `sensor.c` / `sensor.h` | IMX477 I2C register map & init sequence |
| `cyfxgpif2config.h` | GPIF II state machine (generated by GPIF II Designer) |
| `uvc.c` / `uvc.h` | Main application loop, threading, DMA callbacks |
| `cyfxuvcdscr.c` | USB device/configuration/UVC descriptors |
| `cyfxtx.c` | TX descriptors (copy latest from SDK) |
| `cyfx_gcc_startup.S` | GCC startup file (copy latest from SDK) |

**Threading model (GitHub reference implementation):**

| Thread | Responsibility |
|---|---|
| `uvcAppEP0Thread` | USB Control Endpoint 0 — handles UVC Probe/Commit, Brightness, etc. |
| `uvcAppThread` | Application state, sensor start/stop, error recovery |

They communicate via an RTOS Event Group (`glFxUVCEvent`) to prevent the control endpoint from timing out during heavy processing.

**Application thread events:**
```c
CY_FX_UVC_STREAM_ABORT_EVENT | CY_FX_UVC_STREAM_EVENT |
CY_FX_UVC_DMA_RESET_EVENT    | CY_FX_USB_SUSPEND_EVENT_HANDLER,
CYU3P_EVENT_OR_CLEAR
```

**Control thread events:**
```c
CY_FX_UVC_VIDEO_CONTROL_REQUEST_EVENT | CY_FX_UVC_VIDEO_STREAM_REQUEST_EVENT
```

**Robust error handling:**

- **Frame Timer** (`FRAME_TIMER_ENABLE`): if the physical sensor stops sending data (glitch/cable fault), the timer overflows and forces a DMA reset (`CY_FX_UVC_DMA_RESET_FRAME_TIMER_OVERFLOW`).
- **Mac OS workaround** (`glCommitBufferFailureCount`): Mac OS sometimes stops pulling USB packets without issuing a "Stop Streaming" command. The firmware detects full USB buffers, stops the stream, and restarts automatically.

### IMX477 Register Configuration

#### Resolution Pipeline

The sensor operates in a sub-windowed, binned mode to achieve 200 fps:

```
Full sensor array: 4056 × 3040

Step 1 — Analogue readout strip (crop rows via Y_ADD_STA / Y_ADD_END):
  REG_Y_ADD_STA = 528
  REG_Y_ADD_END = 2511
  Active lines   = 2511 − 528 = 1983  →  ~1984 rows read
  Result: 4056 × 1984 analogue buffer

Step 2 — 2×2 Binning:
  Width:  4056 / 2 = 2028
  Height: 1984 / 2 = 992
  Result: 2028 × 992 internal image

Step 3 — Digital ROI crop:
  REG_DIG_CROP_WIDTH  = 640
  REG_DIG_CROP_HEIGHT = 480
  Result: 640 × 480 output over MIPI

Scaling: OFF  (REG_SCALE_MODE=0x00, REG_SCALE_M_MSbit=0x00, REG_SCALE_M_LSB=0x10 → M=16, 16/16=1)
```

#### Frame Rate — 200 fps

Target pixel clock: 200 fps × 557 (frame length) × 2600 (line length) ≈ **290 MHz**

**Internal Pixel Clock (IVT — Logic Speed):**

```
REG_IVT_PREPLLCK_DIV = 0x02  → 27 MHz / 2 = 13.5 MHz
REG_PLL_IVT_MPY      = 0x56  → × 86
REG_IVTPXCK_DIV      = 0x04  → ÷ 4

VCO         = 13.5 × 86 = 1161 MHz
Pixel clock = 1161 / 4  = 290.25 MHz  ✓
```

**MIPI Data Rate (IOP — Wire Speed):**

```
REG_IOP_PREPLLCK_DIV     = 0x02  → 27 / 2 = 13.5 MHz
REG_IOP_MPY              = 0x43  → × 67
REG_REQ_LINK_BIT_RATE    = 0x0405 → 900 Mbps

Actual: 27 / 2 × 67 = 904.5 Mbps per lane
```

**Throughput check:**

```
Total throughput = 290 MHz × 10 bits = 2.9 Gbps
Per lane (4 lanes) = 2.9 / 4 = 725 Mbps/lane

FPGA CSI-2 Rx block settings:
  Line Rate (data rate):          904.5 Mbps
  Clock lane frequency (DDR):     900 Mbps
```

**Frame & Line Length registers:**

```c
{REG_LINE_LEN_MSB,   (2600 >> 8) & 0xFF},   // sensor spends 2600 clocks per line
{REG_LINE_LEN_LSB,    2600       & 0xFF},
{REG_FRAME_LEN_MSB,  (557  >> 8) & 0xFF},   // 557 lines per frame
{REG_FRAME_LEN_LSB,   557        & 0xFF},
```

### GPIF II State Machine

#### Phase 1 — GPIF II Designer GUI

**Step 1 — New project settings**
- Mode: **Synchronous** (driven by external Pixel Clock / PCLK from FPGA)
- Bus width: **32-bit**

**Step 2 — I/O Pin mapping**

| Signal | Direction | Description |
|---|---|---|
| `DQ[0:31]` | Input | Video data |
| `FV` | Ctrl Input | Frame Valid / VSYNC |
| `LV` | Ctrl Input | Line Valid / HSYNC |
| `XCLR` | Ctrl Output | Sensor reset |

**Step 3 — State machine**

```
START (State 0)
  │  (always)
  ▼
WAIT_FOR_FRAME  ──[FV == 0]──▶ (loop)
  │  FV == 1
  ▼
WAIT_FOR_LINE  ──[LV == 0]──▶ (loop)
  │  LV == 1
  ▼
PUSH_DATA  ──[LV == 0]──▶ WAIT_FOR_LINE
  │  Action: IN_DATA (grab DQ[0:31] → DMA socket)
  │  FV == 0
  ▼
END_OF_FRAME
  │  Action: TRIGGER_INTR (notify C code frame is complete)
  └──▶ WAIT_FOR_FRAME
```

**Step 4 — Build:** `Project → Build` generates `cyfxgpif2config.h`. Copy into the FX3 Eclipse IDE project.

#### Phase 2 — Firmware C Code

```c
// 1. Include generated header
#include "cyfxgpif2config.h"

// 2. Initialise PIB (Parallel I/O Block)
CyU3PPibInit(CyTrue, &pibClock);

// 3. Load state machine into GPIF II engine
CyU3PGpifLoad(&CyFxGpifConfig);

// 4. Register interrupt callback (called on TRIGGER_INTR — end of frame)
CyU3PGpifRegisterSMIntrCallback(CyFxGpifCB);

// 5. Start state machine from state 0
CyU3PGpifSMSwitch(GPIF_SM_START_STATE, GPIF_SM_START_STATE,
                  0, GPIF_SM_ALPHA_VALUE, 0);
```

**Overall data flow:**

```
Sensor pixel data
    │  (GPIF II pins, 100 MHz)
    ▼
GPIF II state machine  ──[IN_DATA action]──▶  CY_U3P_PIB_SOCKET_0 (DMA socket)
    │
    ▼  (DMA buffer full)
CyFxUvcApplnDmaCallback  (C code)
    │
    ▼
Prepend 12-byte UVC header
    │
    ▼
USB SuperSpeed endpoint  ──▶  Host PC
```

**UVC descriptors (`cyfxuvcdscr.c`):**

- Resolution: **1280 × 480**
- Frame rate: **200 fps** (frame interval = `10,000,000 / 200 = 50,000` × 100 ns units)
- Colour format: **Mono8 (uncompressed)**
- Recalculate all `Descriptor size` and `Length of this descriptor and all sub-descriptors` fields after any change.
- USB FS / HS config descriptors can set `Number of interfaces = 0x00`; on a USB 3.0 port the FX3 ignores them and uses SuperSpeed descriptors.
